# 懸念点・制限事項

## 作業プロセス

- 手動で行っている作業があるため、100%の正確性を担保できていない
- 作業ミスの介在の余地がある
- 時間と作成者の能力の観点から AI エージェントを利用して実装している
- 適宜確認は行っているつもりではあるが、漏れや抜けなどが存在する可能性がある

## データセット

- datasets/ は MuBench から取得したままであり、一切変更していない
- 元データの正確性や完全性は MuBench に依存する
- datasets/ には 30 プロジェクトが存在するが、src/main/java/ に実装されているのは 11 プロジェクトのみ

## コード修正

- src/main/java/ のコードはビルドを通すため以下のみ修正：
  - パッケージ宣言
  - インポート文
  - public クラス修飾子
- ロジックは変更していないが、この修正がバグの振る舞いに影響を与えないことは保証されていない
- requirements/ は元プロジェクトから依存クラスを抽出しており、抽出漏れや不適切な抽出がある可能性がある
- mocks/ はビルドを通すための簡易実装であり、実際のプロジェクトの動作とは異なる可能性がある

## テスト

- 暫定的に作成したllmによるテストコードの品質については担保されていない
- ソースコードの静的解析によるパターンマッチングでテストしている等、無理やりな実装が存在する（OS依存などの特殊環境への対応）
- 実行時の動作を検証しているわけではない
- misuse はバグを含むため常にコメントアウト

## ビルド

- build.gradle.kts の依存ライブラリバージョンは固定されており、元プロジェクトと異なる可能性がある
- Java 17 でビルドしており、元プロジェクトのコンパイル環境と異なる可能性がある
- **共有時点でのビルド可能性は保証している**（`./gradlew build` が成功することを確認済み）
